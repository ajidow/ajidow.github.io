本站的搭建

发表于 2023-06-05 分类于 一时兴起

Hello World

那么至此，博客已经初具雏形了。取名为猫窝，毕竟是猫头的博客。用的是烂大街的Hexo，部署在github上，因为懒所以选了最轻松的流程。手搓Wordpress不可能的，之前搓过后就再也不想碰了，除非加钱。

接下来要把家里吃灰了十几年的域名挂上去，我bio上的个人网站链接终于要点的动了（

等到下次一时兴起就把网站的i18n做做好，至少把英语页面搞上去。如果有闲心的话说不定还改一改这个主题的css之类的，把暗黑模式的黑全改成#000000。有OLED屏却用假的黑色岂不是很浪费。

接下来说一下搭建过程吧，给自己做个笔记，顺便充当Hello World的功能

安装git

放个链接: git

反正git一直在用，安装git提一嘴即可。接下来所有操作直接在gitbash里面做也行。

安装Node.js

也是老熟人了，去Nodejs官网下载安装一条龙。

安装Hexo

装好Node.js了，他白送你一个包管理器(npm)。

那就直接用npm安装Hexo得了，又是傻瓜式一条龙服务。

npm install -g hexo-cli

搭建博客

找个顺眼的地方放个新建文件夹，接下来在文件夹里生成博客，好处是多个博客可以分开维护。

用hexo init name新建博客，会自动生成一个以name命名的单独的文件夹，博客的内脏全在里面。

进到博客里面，npm install hexo-deployer-git --save装上用于在git部署的模块，npm install hexo-server --save装上用以本地部署，主要是方便调试。

此时hexo s就能在localhost的某个默认端口（应该是4000？）看到裸吊页面了。

接下来又到了我最喜欢的调config时间，博客根目录有个大大的 _config.yml，搞他。

反正英文字母都认识，乱调呗，实在不行看官方文档。

部署到github

config里面有一段关于部署的设置，照着写：

deploy:
  type: git
  repo: https://github.com/Username/Username.github.io.git

之后跑到github新建个仓库，命名严格按照格式Username.github.io

回终端hexo g生成一下，hexo d部署一下就行。

这时候浏览器访问Username.github.io能看到裸的hexo界面。

个性化主题

用的next主题，其实也没个性化到哪里，主要是懒，否则我就手搓css了。

next主题的库我放这里 **链接**，跟着官方文档下一步下一步得了。此处用的是next8。

总结

博客部署开始到结束半小时不到，写文章这么几个字倒是写了一小时，下次引入ChatGPT来帮忙得了，善于利用工具。
